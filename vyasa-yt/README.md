# vyasa-yt

**Mahābhārata → YouTube narration pipeline**

This project scrapes chapter text from [vyas-mahabharat.com](https://vyas-mahabharat.com) as *reference material*, then uses AI to write **original narration scripts** for each chapter, converts them to audio, and assembles portrait-format YouTube videos.

---

## ⚠️ Copyright Notice

- **Translation reference**: Bibek Debroy's Mahābhārata (Penguin Books India) — used as research reference only, **not reproduced** in any video or audio output.  
- The scraped text is stored locally as reference material and is **never** used verbatim in narration scripts, audio, or video content.  
- All narration scripts are original works generated by AI, inspired by the events described in each chapter.  
- Digital source consulted: [vyas-mahabharat.com](https://vyas-mahabharat.com) — credit and gratitude for making this resource available.

---

## About

- **Source**: BORI Critical Edition of the Mahābhārata  
- **Chapters**: 1,795 chapters across 18 parvas (as available on the website)  
- **Format**: Portrait video (1080×1920) with narration audio and subtitles  
- **Purpose**: Educational and personal use  

This channel presents **original narrations** of Mahābhārata events — not readings or paraphrases of any published translation.

## Pipeline Phases

| Phase | Command | Description |
|-------|---------|-------------|
| 0 | `--probe` | Test site structure, verify URLs and CSS selectors |
| 1 | `--scrape` | Scrape chapter text (translation + Sanskrit) as reference |
| 2 | `--scripts` | Generate original narration scripts via Claude API |
| 3 | `--audio` | Convert scripts to MP3 via gTTS |
| 4 | `--video` | Assemble portrait videos (background + audio + subtitles) |
| 5 | `--metadata` | Generate YouTube titles, descriptions, tags via Claude API |

## Quick Start

```bash
# 1. Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set API key (for phases 2 and 5)
export ANTHROPIC_API_KEY="sk-ant-..."

# 4. Run the probe first
python run.py --probe

# 5. Scrape one parvan to test
python run.py --scrape --parvan Adiparvan

# 6. Check status
python run.py --status

# 7. Run the full pipeline for a parvan
python run.py --all --parvan Adiparvan

# 8. Run everything
python run.py --all
```

## CLI Reference

```
python run.py --probe                          # Phase 0: test site structure
python run.py --scrape [--parvan Adiparvan]    # Phase 1: scrape chapters
python run.py --scripts [--parvan Adiparvan]   # Phase 2: narration scripts (Claude)
python run.py --audio [--parvan Adiparvan]     # Phase 3: TTS
python run.py --video [--parvan Adiparvan]     # Phase 4: video assembly
python run.py --metadata [--parvan Adiparvan]  # Phase 5: YouTube metadata
python run.py --all [--parvan Adiparvan]       # All phases (1–5)
python run.py --status                         # Pipeline status table
python run.py --resume --scrape                # Re-scrape only error chapters
python run.py --dry-run --scrape               # Print URLs without fetching
```

## Audio Quality Upgrade

The default TTS uses Google's gTTS (free, functional, robotic).  
For production quality, switch to [ElevenLabs](https://elevenlabs.io):

```bash
pip install elevenlabs
export ELEVENLABS_API_KEY="..."
```

Recommended voices: **Adam** or **Daniel** (deep, authoritative).

Edit `tts.py` to use the ElevenLabs API instead of gTTS.
